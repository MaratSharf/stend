# QR Scanner Интеграция — Станция 6.1

## 📋 Обзор

Подстанция **6.1 QR-scanner** добавлена в систему MES для автоматического сканирования QR-кодов на этапе контроля (станция 6).

## 🏭 Конфигурация станций

```yaml
stations:
  - name: Контроль
    subs:
      - QR-scanner 6.1
```

При перемещении заказа на станцию 6, заказ автоматически направляется на подстанцию 6.1.

## 🚀 Запуск

### 1. MES Production System (основное приложение)

```bash
cd mes_production
python run.py
```

Сервер запустится на **http://localhost:5000**

### 2. QR Scanner Service (сервис сканирования)

```bash
cd services/qr_scanner
pip install -r requirements.txt
python app.py
```

Сервис запустится на **http://localhost:5001**

## 🔧 Настройка окружения

Для QR Scanner сервиса можно задать переменные окружения:

```bash
# URL MES API (по умолчанию http://localhost:5000)
export MES_API_URL=http://localhost:5000

# API ключ для доступа к MES (из config.yaml)
export MES_API_KEY=change-me-to-a-secure-key
```

## 📖 Использование

### Шаг 1: Создание заказа

Через веб-интерфейс MES или API:

```bash
curl -X POST http://localhost:5000/api/orders \
  -H "Content-Type: application/json" \
  -H "X-API-Key: change-me-to-a-secure-key" \
  -d '{
    "batch": "BATCH-A",
    "product_code": "MOLD-220",
    "color": "Красный",
    "quantity": 1
  }'
```

### Шаг 2: Запуск заказа в производство

```bash
curl -X POST http://localhost:5000/api/orders/<order_id>/launch \
  -H "X-API-Key: change-me-to-a-secure-key"
```

### Шаг 3: Перемещение на станцию 6 (QR-scanner)

Передвигайте заказ через станции 1-5, затем:

```bash
curl -X POST http://localhost:5000/api/orders/<order_id>/move \
  -H "X-API-Key: change-me-to-a-secure-key"
```

Заказ попадёт на подстанцию **6.1 QR-scanner**.

### Шаг 4: Сканирование QR-кода

1. Откройте **http://localhost:5001** в браузере
2. **Заказ автоматически загружается** со станции 6.1 (ручной ввод не требуется)
3. Нажмите **"Запустить камеру"**
4. Наведите камеру на QR-код
5. При успешном сканировании результат автоматически сохранится в MES

### Шаг 5: Завершение подстанции

После сканирования нажмите **"Завершить станцию"** — заказ перейдёт на следующую станцию.

## 🔄 Автоматическая загрузка заказа

При открытии страницы QR Scanner автоматически:
1. Запрашивает текущий заказ со станции 6.1 через MES API
2. Отображает информацию о заказе (номер, партия, продукт, цвет, количество)
3. Активирует кнопку запуска камеры

Если станция 6.1 пуста, отображается сообщение "Станция 6.1 пуста. Ожидание заказа..." с кнопкой "Обновить статус".

## 📡 API Endpoints

### QR Scanner Service (порт 5001)

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/` | Главная страница с QR-сканером |
| GET | `/api/current-order` | Получить текущий заказ со станции 6.1 из MES |
| GET | `/api/health` | Проверка работоспособности |
| POST | `/api/scan` | Сохранить результат сканирования (локально) |
| POST | `/api/scan/<order_id>` | Сканирование для заказа с сохранением в MES |
| GET | `/api/scans` | Получить историю сканирований |
| GET | `/api/scans/<order_id>/latest` | Get last scan for order |
| POST | `/api/complete-station` | Завершить подстанцию 6.1 |

### MES API (порт 5000)

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/api/orders/<id>/scan-result` | Сохранить QR результат (из QR Scanner) |
| GET | `/api/orders/<id>/qr-scans` | Получить все сканы для заказа |
| GET | `/api/stations` | Получить статус всех станций (используется для получения заказа 6.1) |

## 💾 База данных

Новая таблица `qr_scans` добавлена автоматически:

```sql
CREATE TABLE qr_scans (
    id SERIAL PRIMARY KEY,
    order_id INTEGER NOT NULL REFERENCES orders(id),
    station_id REAL NOT NULL DEFAULT 6.1,
    qr_data TEXT NOT NULL,
    result TEXT DEFAULT 'OK',
    scanned_at TEXT NOT NULL,
    created_at TEXT NOT NULL
);
```

## 🔄 Жизненный цикла заказа с QR-сканированием

```
[Создание] → buffer 
→ [Запуск] → production 
→ [Станции 1-5] 
→ [Станция 6.1 QR-scanner] → Автоматическая загрузка заказа → Сканирование → Сохранение в БД
→ [Завершение 6.1] 
→ [Станция 7+] 
→ [Завершение]
```

## ✨ Особенности

- 🤖 **Автоматическая загрузка заказа** — заказ подтягивается со станции 6.1 без ручного ввода
- 📷 **Сканирование в реальном времени** — непрерывный анализ видео с камеры
- 💾 **Автосохранение в MES** — каждый скан сохраняется в базу данных
- 🎯 **Привязка к заказу** — результаты сканирования связаны с номером заказа
- 📊 **История сканирований** — все сканы доступны через API и веб-интерфейс
- 🔄 **Автоматическое завершение** — кнопка для перехода на следующую станцию

## 🛠️ Технологии

| Компонент | Технология |
|-----------|------------|
| MES Backend | Flask + Waitress |
| QR Scanner | Flask + Waitress + jsQR |
| База данных | PostgreSQL / SQLite |
| Фронтенд | HTML5 + JavaScript + MediaDevices API |

## 📝 Примечания

- Требуется HTTPS или localhost для доступа к камере (политика браузера)
- Работает в современных браузерах (Chrome, Firefox, Edge, Safari)
- API ключ должен соответствовать значению в `config.yaml`
- Подстанция 6.1 должна быть завершена перед переходом на следующую станцию
- **Ручной ввод номера заказа больше не требуется** — заказ загружается автоматически из MES
