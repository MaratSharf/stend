# 🏭 MES Production System

**Manufacturing Execution System (MES)** — веб-приложение для управления производственными заказами и отслеживания прохождения заказов через рабочие станции.

## 📋 Описание

Система позволяет:

- ✅ **Создавать пакеты заказов** с автогенерацией уникальных номеров
- ✅ **Запускать заказы в производство** на первую станцию
- ✅ **Отслеживать перемещение заказов** между рабочими станциями
- ✅ **Автоматически завершать заказы** после прохождения всех станций
- ✅ **Просматривать статистику производства** в реальном времени
- ✅ **Управлять статусом заказов** (буфер, производство, завершено, отменено)

### 🔄 Производственные станции

Система включает 10 рабочих станций:

| № | Станция | № | Станция |
|---|---------|---|---------|
| 1 | Приёмка | 6 | Контроль |
| 2 | Сортировка | 7 | Тестирование |
| 3 | Подготовка | 8 | Упаковка |
| 4 | Сборка | 9 | Маркировка |
| 5 | Пайка | 10 | Отгрузка |

## 🚀 Установка и запуск

### Предварительные требования

- Python 3.10 или выше
- pip (менеджер пакетов Python)

### Шаг 1: Установка зависимостей

```bash
pip install -r requirements.txt
```

### Шаг 2: Настройка конфигурации (опционально)

Откройте `config.yaml` и измените параметры:

```yaml
database:
  path: data/mes.db          # Путь к базе данных
server:
  host: 0.0.0.0              # Адрес сервера
  port: 5000                 # Порт сервера
auth:
  api_keys:
    - ваш-секретный-ключ     # Ключ для API
logging:
  level: INFO
  path: data/logs
```

### Шаг 3: Запуск приложения

```bash
python run.py
```

Приложение запустится и будет доступно по адресу:
- **http://localhost:5000**

### Шаг 4: Остановка

Нажмите **Ctrl+C** в терминале.

## 📡 API Endpoints

### Заказы

| Метод | Endpoint | Доступ | Описание |
|-------|----------|--------|----------|
| GET | `/api/orders` | Публичный | Все заказы (`?status=buffer/production/completed/cancelled`) |
| POST | `/api/orders` | 🔒 Ключ | Создать заказы |
| POST | `/api/orders/<id>/launch` | 🔒 Ключ | Запустить в производство |
| POST | `/api/orders/<id>/move` | 🔒 Ключ | Переместить на след. станцию |
| POST | `/api/orders/<id>/complete` | 🔒 Ключ | Завершить вручную |
| POST | `/api/orders/<id>/cancel` | 🔒 Ключ | Отменить заказ |

### Станции и статистика

| Метод | Endpoint | Доступ | Описание |
|-------|----------|--------|----------|
| GET | `/api/stations` | Публичный | Статус всех станций |
| GET | `/api/statistics` | Публичный | Статистика производства |

### Создание заказов (пакетная генерация)

**Формат запроса:**
```json
{
  "batch": "BATCH-A",
  "product_code": "MOLD-220",
  "color": "Красный",
  "quantity": 3
}
```

**Формат ответа:**
```json
{
  "success": true,
  "orders": [
    {
      "id": 1,
      "batch": "BATCH-A",
      "order_number": "ORD-4521-789",
      "product_code": "MOLD-220",
      "color": "Красный",
      "quantity": 1,
      "status": "buffer"
    }
  ],
  "count": 3,
  "message": "Created 3 order(s)"
}
```

### Автогенерация номеров заказов

Формат: `ORD-<последние-4-цифры-timestamp>-<3-случайные-цифры>`

Примеры: `ORD-4521-789`, `ORD-4522-456`, `ORD-4523-123`

Каждый созданный заказ имеет `quantity: 1`. Поле `quantity` в запросе означает **количество заказов для генерации**.

### Примеры curl

**Создание заказов:**
```bash
curl -X POST http://localhost:5000/api/orders \
  -H "Content-Type: application/json" \
  -H "X-API-Key: ваш-ключ" \
  -d '{"batch":"B1","product_code":"P1","color":"Red","quantity":3}'
```

**Получение заказов:**
```bash
curl http://localhost:5000/api/orders
```

**Запуск заказа:**
```bash
curl -X POST http://localhost:5000/api/orders/1/launch \
  -H "X-API-Key: ваш-ключ"
```

## 🔐 Аутентификация

- **GET-запросы** — публичные (чтение)
- **POST-запросы** — требуют API-ключ в заголовке `X-API-Key`

Ключи настраиваются в `config.yaml` в секции `auth.api_keys`.
Если список ключей пуст — авторизация отключена.

Фронтенд автоматически подставляет ключ из `<meta name="api-key">` тега.

## 📊 Статусы заказов

| Статус | Описание |
|--------|----------|
| `buffer` | Заказ создан, ожидает запуска |
| `production` | Заказ в производстве |
| `completed` | Заказ завершён |
| `cancelled` | Заказ отменён |

## 🗄️ База данных

При первом запуске автоматически создаётся база данных `data/mes.db`.

### Таблицы

| Таблица | Описание |
|---------|----------|
| `orders` | Заказы с отслеживанием статуса |
| `stations` | 10 рабочих станций с текущими заказами |
| `station_log` | Журнал прохождения станций |

### Особенности

- **Внешние ключи** включены (`PRAGMA foreign_keys = ON`)
- **Транзакции** с rollback для многошаговых операций
- **Уникальные номера** заказов (timestamp + random)
