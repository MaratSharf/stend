# MES Production System

**Manufacturing Execution System (MES)** — веб-приложение для управления производственными заказами и отслеживания их прохождения через рабочие станции с поддержкой подстанций, аутентификации и ролевой модели.

## Возможности

- ✅ **Пакетное создание заказов** с автогенерацией уникальных номеров (`ORD-0001`, `ORD-0002`, ...)
- ✅ **Аутентификация** — логин/пароль через Flask-Login
- ✅ **Роли пользователей** — `admin`, `operator`, `viewer` с иерархией прав
- ✅ **Подстанции** — основные станции могут иметь дочерние (1.1, 1.2, 3.1, ...)
- ✅ **Несколько заказов на станции** — параллельная обработка
- ✅ **Автоматическое завершение** заказов после прохождения всех станций
- ✅ **Статистика производства** в реальном времени
- ✅ **SVG-карта производства** — визуализация потока заказов по станциям
- ✅ **Тёмная/светлая тема**

## Технологии

| Категория | Технология |
|-----------|------------|
| Язык | Python 3.10+ |
| Веб-фреймворк | Flask >= 3.0.0 |

## Установка и запуск

### Предварительные требования

- Python 3.10 или выше

### Шаг 1: Создание виртуальной среды

```bash
python -m venv venv
```

### Шаг 2: Активация виртуальной среды

**Windows:**
```bash
venv\Scripts\activate
```

**Linux/macOS:**
```bash
source venv/bin/activate
```

### Шаг 3: Установка зависимостей

```bash
pip install -r mes_production/requirements.txt
```

### Шаг 4: Запуск приложения

```bash
python mes_production\run.py
```

Приложение запустится на **http://localhost:5000**

### Шаг 5: Вход в систему

- **Логин:** `admin`
- **Пароль:** `admin`

> ⚠️ Смените пароль администратора после первого входа через страницу `/users`.

### Остановка

Нажмите **Ctrl+C** в терминале.

## Страницы приложения

| Страница | URL | Описание |
|----------|-----|----------|
| Вход | `/login` | Аутентификация пользователя |
| Заказы | `/` | Список всех заказов с созданием и фильтрацией |
| Карта станций | `/tracking` | Карточки станций — клик открывает модалку с деталями заказов |
| Станция | `/station` | Выбор станции из dropdown → таблица заказов |

| Пользователи | `/users` | Управление пользователями (только admin) |

## Конфигурация

Все настройки в `mes_production/config.yaml`:

```yaml
database:
  path: data/mes.db

server:
  host: 0.0.0.0
  port: 5000

stations:
  - name: Приёмка
    subs:
      - Приёмка 1.1
      - Приёмка 1.2
  - name: Сортировка
  - name: Подготовка
    subs:
      - Подготовка 3.1
  - name: Сборка
  # ... остальные станции

auth:
  api_keys:
    - ваш-секретный-ключ

logging:
  level: INFO
  path: data/logs
```

## Роли пользователей

| Роль | Доступ |
|------|--------|
| **viewer** | Просмотр заказов, станций, статистики |
| **operator** | viewer + создание/запуск/перемещение/завершение заказов |
| **admin** | operator + управление пользователями |

## API

### Заказы

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/orders` | Все заказы (`?status=buffer/production/completed/cancelled`) |
| POST | `/api/orders` | Создать заказы |
| POST | `/api/orders/<id>/launch` | Запустить в производство |
| POST | `/api/orders/<id>/move` | Переместить на след. станцию |
| POST | `/api/orders/<id>/complete` | Завершить вручную |
| POST | `/api/orders/<id>/cancel` | Отменить заказ |

### Станции и статистика

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/stations` | Все станции с текущими заказами |
| GET | `/api/statistics` | Статистика производства |

### Аутентификация API

API принимает **либо** сессию браузера, **либо** API-ключ в заголовке `X-API-Key`:

```bash
# Через API-ключ
curl -X POST http://localhost:5000/api/orders \
  -H "Content-Type: application/json" \
  -H "X-API-Key: ваш-ключ" \
  -d '{"batch":"B1","product_code":"P1","color":"Red","quantity":3}'

# Через сессию (браузер отправляет cookie автоматически)
```
