# QR Scanner Service с интеграцией MES Production

Веб-сервис для считывания QR-кодов с камеры ноутбука с сохранением результатов в базу данных MES Production System.

## 🚀 Запуск

### 1. Установить зависимости

```bash
cd services/qr_scanner_db
pip install -r requirements.txt
```

### 2. Запустить сервис

```bash
python app.py
```

Сервис запустится на **http://localhost:5001**

## 📖 Использование

### Автоматическое сканирование

1. Откройте браузер и перейдите по адресу `http://localhost:5001`
2. Нажмите кнопку **"Запустить камеру"**
3. Разрешите доступ к камере в браузере
4. Направьте камеру на QR-код
5. При успешном сканировании результат автоматически сохраняется в БД

### Ручное сохранение

1. Отсканируйте QR-код (или введите данные вручную)
2. Заполните дополнительные поля (при необходимости):
   - **ID заказа** — ссылка на заказ из системы MES
   - **ID станции** — номер рабочей станции (например, 1.1, 3.1)
3. Нажмите кнопку **"Сохранить вручную"**

### Просмотр истории

- Все сканирования отображаются в правой панели
- Показывается статистика (всего / за сегодня)
- Можно удалить отдельные записи кнопкой "×"
- Данные хранятся в таблице `qr_scans` базы данных `mes.db`

## ✨ Функции

- 📷 **Сканирование в реальном времени** — непрерывный анализ видео с камеры
- 💾 **Автоматическое сохранение в БД** — все сканирования записываются в `mes.db`
- 🔗 **Связь с заказами MES** — возможность указать `order_id` и `station_id`
- 📊 **Статистика** — общее количество и сканирования за сегодня
- 📋 **История** — просмотр и удаление записей
- 📱 **Адаптивный интерфейс** — работает на десктопе и мобильных устройствах
- 🌓 **Тёмная тема** — современный дизайн

## 🛠️ Технологии

- **Backend:** Python Flask + Waitress
- **Frontend:** HTML5 + JavaScript
- **QR декодирование:** jsQR (библиотека JavaScript)
- **База данных:** SQLite (mes.db из MES Production System)
- **Доступ к камере:** MediaDevices API

## 🌐 API Endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/` | Главная страница с QR-сканером |
| GET | `/api/health` | Проверка работоспособности |
| POST | `/api/scan` | Сохранить QR-код в БД |
| GET | `/api/scans` | Получить историю сканирований |
| DELETE | `/api/scans/<id>` | Удалить запись сканирования |

### Примеры запросов

**Сохранение QR-кода:**
```bash
curl -X POST http://localhost:5001/api/scan \
  -H "Content-Type: application/json" \
  -d '{"qr_data": "ORD-0042-ABC123", "order_id": 42, "station_id": 1.1}'
```

**Получение истории:**
```bash
curl http://localhost:5001/api/scans?limit=50
```

**Фильтр по заказу:**
```bash
curl http://localhost:5001/api/scans?order_id=42
```

## 🗄️ База данных

Сервис автоматически создаёт таблицу `qr_scans` в базе `data/mes.db`:

```sql
CREATE TABLE qr_scans (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    qr_data TEXT NOT NULL,
    scan_time TEXT DEFAULT CURRENT_TIMESTAMP,
    order_id INTEGER REFERENCES orders(id),
    station_id REAL,
    notes TEXT
)
```

## ⚙️ Настройки

Порт сервера можно изменить в файле `app.py`:

```python
serve(app, host='0.0.0.0', port=5001, threads=4)
```

Путь к базе данных MES:
```python
MES_DB_PATH = os.path.join(os.path.dirname(__file__), '..', '..', 'data', 'mes.db')
```

## 📝 Примечания

- Требуется HTTPS или localhost для доступа к камере (политика браузера)
- Работает в современных браузерах (Chrome, Firefox, Edge, Safari)
- База данных `mes.db` должна находиться в `data/mes.db` относительно корневого каталога проекта
- Таблица `qr_scans` создаётся автоматически при первом запуске

## 🔗 Интеграция с MES

Сервис использует общую базу данных с MES Production System. Это позволяет:
- Ссылаться на существующие заказы через `order_id`
- Указывать рабочую станцию через `station_id`
- Агрегировать данные сканирований с другими данными производства
