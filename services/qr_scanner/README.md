# QR Scanner Service

Веб-сервис для считывания QR-кодов с камеры ноутбука. Интегрируется с MES Production System.

## 🚀 Запуск

### 1. Установить зависимости

```bash
cd services/qr_scanner
pip install -r requirements.txt
```

### 2. Запустить сервис

```bash
python app.py
```

Сервис запустится на **http://localhost:5001**

## 📖 Использование

1. Откройте браузер и перейдите по адресу `http://localhost:5001`
2. Нажмите кнопку **"Запустить камеру"**
3. Разрешите доступ к камере в браузере
4. Направьте камеру на QR-код
5. При успешном сканировании результат отобразится на экране
6. История сканирования сохраняется в браузере

## ✨ Функции

- 📷 **Сканирование в реальном времени** — непрерывный анализ видео с камеры
- 📋 **История сканирования** — последние 10 отсканированных кодов сохраняются
- 🌓 **Тёмная тема** — современный дизайн с тёмным фоном
- 📱 **Адаптивный интерфейс** — работает на десктопе и мобильных устройствах
- 🔄 **Автоматическое сохранение** — история сохраняется в localStorage браузера
- 🔗 **Интеграция с MES** — отправляет результаты сканирования в основное приложение

## 🛠️ Технологии

- **Backend:** Python Flask + Waitress
- **Frontend:** HTML5 + JavaScript (ES6+)
- **QR декодирование:** jsQR (библиотека JavaScript)
- **Доступ к камере:** MediaDevices API
- **Canvas оптимизация:** `willReadFrequently = true` для частых операций `getImageData`

## 🌐 API Endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/` | Главная страница с QR-сканером |
| GET | `/api/health` | Проверка работоспособности |
| GET | `/api/current-order` | Получить текущий заказ со станции 6.1 (из MES) |
| POST | `/api/scan` | Сохранить результат сканирования (локально) |
| POST | `/api/scan/<order_id>` | Сканирование для заказа с сохранением в MES |
| GET | `/api/scans` | Получить историю сканирований |
| GET | `/api/scans/<order_id>/latest` | Получить последний скан для заказа |
| POST | `/api/complete-station` | Завершить подстанцию 6.1 |

## ⚙️ Настройки

### Порт сервера

Можно изменить в файле `app.py`:

```python
serve(app, host='0.0.0.0', port=5001, threads=4)
```

### Переменные окружения

```bash
# URL MES API (по умолчанию http://localhost:5000)
export MES_API_URL=http://localhost:5000

# API ключ для доступа к MES (из config.yaml)
export MES_API_KEY=change-me-to-a-secure-key
```

## 📝 Примечания

- **Требуется HTTPS или localhost** для доступа к камере (политика браузера)
- **Работает в современных браузерах** (Chrome, Firefox, Edge, Safari)
- **История сканирования** хранится только в локальном браузере (localStorage)
- **Оптимизация Canvas:** используется `willReadFrequently = true` для улучшения производительности при частых операциях `getImageData`

## 🔧 Устранение проблем

### Ошибка доступа к камере

**Причина:** Браузер блокирует доступ к камере на HTTP (небезопасном) протоколе.

**Решение:**
- Запустите сервис на `localhost` (работает по умолчанию)
- Или настройте HTTPS для удалённого доступа

### Предупреждение Canvas2D

Если видите предупреждение:
```
Canvas2D: Multiple readback operations using getImageData are faster with the willReadFrequently attribute set to true
```

**Решение:** Уже реализовано в коде через `canvas.getContext('2d', { willReadFrequently: true })`.

### Сканирование не работает

**Проверьте:**
1. Камера подключена и работает
2. Разрешение доступа к камере в браузере
3. Освещение и контрастность QR-кода
4. Расстояние до камеры (оптимально 20-40 см)

## 🔄 Интеграция с MES Production System

QR Scanner работает как отдельный сервис на порту 5001 и взаимодействует с основным приложением (порт 5000):

1. **Автозагрузка заказа:** При открытии страницы сервис автоматически получает заказ со станции 6.1 через MES API
2. **Сохранение результатов:** После сканирования результат отправляется в MES через `/api/orders/<id>/scan-result`
3. **Завершение подстанции:** Кнопка "Завершить станцию" перемещает заказ на следующую станцию
